# Dev Log — 2026-03-08

Digest quality polish sprint. No new features — pure output quality work. Pipeline now produces a structurally clean, non-redundant, well-formatted digest that matches the target newsletter design. First HTML email template created for Portfolio Pulse.

## Context Going In

Pipeline was running end-to-end but digest quality was poor: 35+ news bullets, duplicate facts across every section, truncated driver sentences in the Movers table, small insider sales cluttering the output ($70K CAO trades getting the same billing as $36M CFO sales), and no hard enforcement on any of the LLM limits. The Movers table also had a double-dash display bug (`- -1.5%`).

---

## Changes Made (in order)

### 1. Prompt-level quality rules — `model/model.py`

Added five new rules to `build_eod_summary_prompt()`:

**KEY MARKET INSIGHTS LIMIT** — Hard cap at 15 bullets, one per ticker. LLM told to rank by: earnings beats > analyst upgrades/PTs > product launches > general commentary, and to drop the lowest-signal tickers if over 15.

**SECTION ROLES** — Explicit instruction that the Movers table owns the price movement + primary driver. Key Market Insights must cover *additional* facts not already in the Movers driver. News That Mattered must cover articles whose key point wasn't already in Key Insights. This is the core deduplication mechanism.

**INSIDER TRANSACTIONS** — LLM instructed to not mention insider sales/purchases under $1M. Concrete examples given: exclude a COO selling 629 shares totaling $400K; include a CFO selling $36M. LLM prompt alone isn't bulletproof (it was the fallback for articles that slip through the noise filter).

**NEWS THAT MATTERED cap** — Hard cap at 10 items, with explicit rule to skip articles already covered in Key Market Insights.

**Deduplication** — Strengthened the existing rule to explicitly say "do not restate the same fact in different words."

---

### 2. Movers table: ▲/▼ arrows — `pipeline/run_test_pipeline.py`

**Bug:** `arrow = "- "` combined with `{pct:.1f}%` which is already negative → rendered as `- -1.5%`.

**Fix:** Changed to `arrow = "▲" if pct >= 0 else "▼"` with a separate `sign = "+"` prefix for positive values. Now renders `▲ +3.1%` / `▼ -1.5%`.

---

### 3. Scraping deduplication — `pipeline/run_test_pipeline.py`

**Bug:** KO article body text contained repeated sentences from pagination artifacts. The LLM reproduced them verbatim: `"th organic revenues growing 5% year-over-year."` appeared three times in the output.

**Fix:** In `_enrich_articles_with_full_text()`, after fetching the scraped body, split on newlines and remove consecutive duplicate lines before appending. Conservative approach — only removes exact consecutive repeats, not all duplicates. Preserves legitimately repeated paragraphs in different sections of the article.

---

### 4. Key Market Insights hard cap — `pipeline/run_test_pipeline.py`

**Problem:** LLM prompt said "maximum 15 bullets" but the LLM wrote 19 in the first run, 19 in the second. Prompt rules are not reliable for hard numeric limits.

**Fix:** New `_cap_key_insights(eod_text, limit=15)` function. After the LLM returns the EOD summary, this function:
1. Regex-finds the "Key Market Insights" section and extracts all bullet lines
2. If count > 15, slices to first 15 (LLM already ranks by importance, so top bullets are best)
3. Splices the capped block back into the full text without touching other sections
4. Logs how many bullets were dropped

Applied immediately after `get_end_of_day_summary()` returns, before saving the digest.

---

### 5. Sentence-boundary truncation — `pipeline/run_test_pipeline.py`

**Bug:** Movers table driver text was sliced at exactly 120 chars, often mid-sentence: `"...with EPS of"`. Unreadable.

**Fix:** New `_truncate_at_sentence(text, limit=200)` helper. Finds the last `". "` occurrence within the first `limit` chars. If found past the midpoint of the limit, cuts there. Falls back to hard truncation with `…` only if no sentence boundary exists (handles long single-sentence insights). Limit raised from 120 → 200 to give LLM more room to write a complete sentence.

---

### 6. Noise filter: insider sales under $1M — `news/noise_filter.py`

**Two new patterns added.**

**Pattern 1 — "Insider Selling:" prefix with small dollar amount:**
```
\binsider\s+(?:selling|sale|transaction)s?\b
(?!.*\b(?:million|billion)\b)
.{0,150}\$\d{1,3},\d{3}(?:\.\d+)?(?![\d,])
```
Matches titles like `"Insider Selling: DraftKings CAO Sells $70,806.48 in Stock"`. Dollar amount pattern `\$\d{1,3},\d{3}` (exactly one comma group) covers $1,000–$999,999. Negative lookahead `(?![\d,])` prevents matching mid-number (won't catch `$36,471,323`). Skips titles with "million"/"billion".

**Pattern 2 — Role-based insider sale without "insider" keyword:**
```
\b(?:CEO|CFO|COO|CTO|CMO|CRO|CAO|SVP|EVP|VP|President|Director|Chairman)\b
.{0,80}\bsells?\b
(?!.*\b(?:million|billion)\b)
.{0,80}\$\d{1,3},\d{3}(?:\.\d+)?(?![\d,])
```
Catches `"Verizon (NYSE:VZ) SVP Sells $428,450.00 in Stock"` — same amount logic, but anchored to executive role keywords instead of "insider". Does NOT match `buys?` — insider buys are always kept regardless of size (a buy is an unambiguous bullish signal; a small sell is routine diversification).

**Result:** 2 articles noise-skipped at query time in second test run (up from 1).

## Challenges Encountered

**1. LLM not respecting hard limits in prompts**
Adding "maximum 15 bullets" to the prompt did not work. The LLM produced 19 in both runs before the Python-level cap was added. Lesson: for any hard numeric constraint, always enforce it in code post-processing, not just in the prompt. Prompts are for quality guidance; Python is for hard rules.

**2. Noise filter dollar amount matching — avoiding partial matches**
The regex `\$\d{1,3},\d{3}` (one comma group) matches amounts up to $999,999. But `$36,471,323` also starts with `$36,471` — a naive pattern would match mid-number. The negative lookahead `(?![\d,])` after the pattern prevents the match from succeeding if followed by more digits or a comma, ensuring the full number is evaluated before deciding. Took a few iterations to get right.

**3. Scraping artifacts vs. legitimate repeated text**
The KO article had `"th organic revenues growing 5% year-over-year."` repeated three times — clearly a rendering artifact from the scraper splitting a sentence mid-word across pagination. But a blanket "deduplicate all sentences" approach could silently drop legitimate repetitions in long articles. Settled on consecutive-line dedup only: if the exact same line appears back-to-back, drop the duplicate. Handles pagination artifacts, preserves intentional structure.

**4. Movers driver misattributions**
In Run 1, META's Movers driver read: "In Q4 2025, Berkshire Hathaway reduced its stake in Meta Platforms." This is a BRK-A fact the insights LLM attributed to META's price movement. Root cause: the insights prompt has no instruction about insider transaction materiality. The INSIDER TRANSACTIONS rule only exists in the EOD summary prompt. Fix deferred to next session.

That function has no materiality filter, so sub-$1M insider sales end up as Movers drivers (META COO $400K). Fix: add same rule to `build_insight_prompt()`.

This causes high-signal tickers like DKNG (Director $2.185M insider buy) to get cut. `"Meta Platforms (NASDAQ:META) Stock Price Down 1.1% Following Insider Selling"` — no dollar amount in title, so neither noise filter pattern matches. The article passes ingest and relevance filter, and the LLM uses it in "News That Mattered." Need a pattern that catches `"Following Insider Selling"` suffix when the title contains no material dollar amount (no "million"/"billion").
